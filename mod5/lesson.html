Module 5: Web Accessibility

Introduction to Web Design Tools and Software


Lesson Topics

Web accessibility refers to the inclusive practice of designing websites and digital content that is easily navigated and understood by all users, including those with disabilities. It aims to ensure that people with disabilities can perceive, understand, navigate, and interact with the web and that they can contribute to the web.

The Importance:

  • Accessibility is a fundamental human right, as recognized by the United Nations Convention on the Rights of Persons with Disabilities.
  • It promotes inclusivity and diversity, enabling people with disabilities to access the same information and services as everyone else.
  • It enhances user experience for all users, including those without disabilities, by improving usability and clarity of content.

impact of inaccessible websites on users with disabilities:

Inaccessible websites can create significant barriers for users with disabilities, making it difficult or impossible for them to access information or services online.

Examples of impact:

  • Visually impaired users may be unable to use screen readers to access content if proper alternative text is not provided for images.
  • Users with mobility impairments may struggle to navigate websites that are not keyboard accessible.
  • People with cognitive disabilities may struggle to understand complex navigation or content that is not structured clearly.

The impact goes beyond individual users, affecting their ability to participate in education, employment, and social activities online.

Highlight the benefits of creating accessible websites for all users:

  • Improved usability: Accessible design principles often lead to better overall usability for all users, not just those with disabilities.
  • Enhanced SEO: Many accessibility practices, such as providing descriptive text for images, can improve a website's search engine optimization (SEO).
  • Legal compliance: In many countries, there are legal requirements for websites to be accessible, making it important for organizations to ensure compliance.
  • Positive reputation: Demonstrating a commitment to accessibility can enhance an organization's reputation and appeal to a wider audience.
  • Future-proofing: Designing with accessibility in mind can help ensure that websites remain usable as technology evolves.

Key Accessibility Guidelines and Standards

Web Content Accessibility Guidelines (WCAG): Developed by the World Wide Web Consortium (W3C), WCAG provides a set of guidelines for making web content more accessible to people with disabilities. It is organized into four principles: perceivable, operable, understandable, and robust.

Section 508: Section 508 of the Rehabilitation Act requires federal agencies to make their electronic and information technology accessible to people with disabilities. It sets forth accessibility standards that are similar to WCAG but specifically apply to federal agencies in the United States.

Principles of Accessible Design

Perceivable: Information and user interface components must be presentable to users in ways they can perceive. This includes providing text alternatives for non-text content (such as images or videos), making sure content can be presented in different ways (e.g., larger text, simpler layout), and ensuring content is distinguishable and easy to perceive (e.g., using sufficient color contrast).

Operable: User interface components and navigation must be operable. This means users should be able to interact with all controls and interactive elements using either a keyboard or assistive technologies. It also includes providing users with enough time to read and use content, avoiding content that causes seizures, and designing for compatibility with assistive technologies.

Understandable: Information and the operation of the user interface must be understandable. This includes making text readable and understandable, ensuring content appears and operates in predictable ways, and helping users avoid and correct mistakes.

Robust: Content must be robust enough that it can be interpreted reliably by a wide variety of user agents, including assistive technologies. This means using valid and well-structured code (e.g., HTML) and ensuring compatibility with current and future technologies.

By adhering to these principles and guidelines, web developers and designers can create websites that are more inclusive and accessible to all users, including those with disabilities.

Creating Accessible Websites

Alternative Text for Image

Provide descriptive alt text for images so that users with visual impairments can understand the content of the image. alt text should be concise and descriptive, conveying the purpose or information conveyed by the image.

<img src="" alt="img of ...">

Keyboard Navigation

Ensure that all functionality of the website can be accessed and operated using a keyboard alone, without requiring a mouse. This is essential for users who cannot use a mouse, including those with mobility impairments.

Tab Navigation

Pressing the Tab key allows you to navigate through interactive elements on the page, such as links, buttons, and form fields. Each element should receive focus in a logical order, ensuring a smooth and intuitive navigation experience.

Dropdown Menus

When you encounter a dropdown menu, pressing the Tab key should allow you to open the menu and navigate its options using the arrow keys. You should also be able to close the menu using the Escape key.

Interactive Maps

If the website includes an interactive map, you should be able to navigate and interact with it using only your keyboard. The map should be accessible and allow you to zoom in and out, pan, and select locations.

Form Fields

When filling out a form, you should be able to navigate between form fields using the Tab key. Required fields should be clearly indicated, and error messages should be displayed in a way that is accessible to keyboard users.

Modal Windows

If a modal window (such as a popup dialog) appears on the page, you should be able to interact with it using only your keyboard. Pressing the Tab key should allow you to navigate through the modal's elements, and pressing the Escape key should close the modal.

Skip Links

The website should include a "Skip to main content" link at the top of the page that allows you to bypass repetitive navigation links and go directly to the main content area. This helps users who navigate using a keyboard to quickly access the content they are interested in.

Color Contrast

Use sufficient color contrast between text and background colors to ensure readability for users with low vision or color blindness. Tools like color contrast checkers can help you determine if your color choices meet accessibility standards.

Accessible Forms

Ensure that all form elements (e.g., input fields, checkboxes, radio buttons) are properly labeled and accessible to screen readers. Provide clear instructions and error messages for form validation.

Video and Audio Accessibility

Provide captions for videos and transcripts for audio content to make them accessible to users who are deaf or hard of hearing. Provide controls for users to adjust volume and playback speed.

Accessible Tables

Use proper table markup (<tablegt;, <thgt;, <tdgt;) and provide table headers (<thgt;) to make tables accessible to screen readers. Avoid using tables for layout purposes.

Focus Styles

Ensure that interactive elements (like links and buttons) have a visible focus indicator when they are in focus. This helps users who navigate using a keyboard or assistive technology see where they are on the page.

Demonstration of Accessibility Tools

Screen Readers

Demonstrate how a screen reader (such as NVDA, VoiceOver, or JAWS) reads aloud the content of a webpage and how users can navigate through the page using keyboard shortcuts.

Accessibility Validators

Introduce tools like WAVE (Web Accessibility Evaluation Tool) or Axe that can automatically scan a webpage for accessibility issues and provide suggestions for improvement.

By following these best practices and using accessibility tools, developers can create websites that are more inclusive and accessible to all users, including those with disabilities.
